  42 \section{Prerequisites}
  43 A quick list before getting into details:
  44 \begin{enumerate}
  45 \item Java 6 or 7 (8 won't work yet)
  46 \item Python 2.x
  47 \item Eclipse 3.7 (Indigo) or newer
  48 \item Wesnoth 1.9.x, trunk or newer
  49 \end{enumerate}
  50 
  51 Now for getting all those items:
  52 \begin{enumerate}
  53 \item The plugin runs on the following platforms: Windows 32/64 bit, Linux 32/64 bit and Mac OS X 64bit. Note that \textbf{Mac OS X 32 bit} is \textbf{not supported}. If you want to know why, please consult the Frequently Asked Questions section.
  54 \item Download and install Oracle's Java Version 7 (Java SE7): \href{http://www.oracle.com/technetwork/java/javase/downloads/jdk7-downloads-1880260.html}{Download JDK}
  55 
  56 \textit{Note:} The plugin uses Java SE6/SE7, thus older versions (like 1.4 or 1.5) won't work with it.\\
  57 \textit{Note:} Please double check the java installed on your system. On some machines there is the OpenJDK or other Java versions. Just use Oracle's so there will be no problems.
  58 
  59 \item Download and install Python 2.x:
  60  \begin{enumerate}
  61    \item \textbf{Windows:} Download and install it from here: \href{http://python.org/download/}{Download Python} , selecting a 2.x version
  62    \item \textbf{Linux:} Use the distribution's package manager for installing it.
  63    \item \textbf{Mac:} Download and install it from here: \href{http://python.org/download/}{Download Python} , selecting a 2.x version
  64    \item For other operating systems, check the guide over here: \href{http://wiki.python.org/moin/BeginnersGuide/Download}{Python download and install}
  65   \end{enumerate}
  66  \textit{Note:} Please ensure you install the 2.x version. Version 3.x is \textbf{not} yet supported by the Wesnoth's WML Tools.
  67 
  68 \item Download ``Eclipse" (The download links are in the right. Please ensure you are downloading at least the \textbf{3.7} version, otherwise the plugin will not work. Also, \textbf{don't} download a \textbf{64 bit version} unless you are sure you have Java JDK on 64 bit. If you are unsure, select the 32 bit version):  \href{http://eclipse.org/downloads/packages/eclipse-rcp-and-rap-developers/indigor}{Download Eclipse for RCP and RAP Developers}
  69 
  70 \item Extract the downloaded archive in a known location and launch the executable (eclipse / eclipse.exe)
  71 
  72 \item You will need to have a wesnoth version 1.9.x, trunk or newer, in order for the plugin's features to correctly work.
  73 \end{enumerate}

  75 \section{Setup the environment}
  76 After you got all the prerequisites, you need to setup the eclipse installation.
  77 
  78 \subsection{Target Platform setup}
  79 \begin{enumerate}
  80 \item Download the ``eclipse\_3.7.zip" archive, that contains an already existing Eclipse SDK installation and deltapack (used for building against other OSes than the native one). The link for the download is: \href{http://sourceforge.net/projects/wesnoth/files/wesnoth-umcplugin/build\_utils/eclipse\_3.7.zip/download}{Download}
  81 
  82 \textit{Note:} You can generate the archive contents yourself by downloading the Eclipse SDK and its corresponding deltapack from \url{http://download.eclipse.org/eclipse/downloads/}.
  83 
  84 \item Extract the archive somewhere.
  85 
  86 \item Go to menu: Window -$>$ Preferences-$>$ Plug-in Development-$>$ Target Platform
  87 
  88 \item Select the \textbf{Running Platform (Active)} item and press ``Edit..." in the left buttons lane.
  89 
  90 \item A new window with tabs will be opened. Select the ``Locations" tab, and in the right buttons lane, select ``Add..."
  91 
  92 \item Select ``Directory" in the new windows, then press Next.
  93 
  94 \item Browse to the previous location where you have extracted the ``eclipse\_3.7.zip" archive.
  95 
  96 \item Press Finish and OK until you exist all windows.
  97 \end{enumerate}

  99 \subsection{Installing Xtext}
 100 \begin{enumerate}
 101 \item Open up the ``Install new software" menu, and select from the list: ``Indigo - http://download.eclipse.org/releases/indigo" (if using a newer version, just select the url that starts with \emph{http://download.eclipse.org/releases})
 102 
 103 \item In the list populated with items, select from the ``Modelling" category: ``Xtext SDK", and install it. Restart eclipse after that.
 104 
 105 \item For the plugin's source, checkout the folder ``umc\_dev" from
 106   the Battle for Wesnoth repository.
 107 
 108 \item In Eclipse, right click in \textbf{Package Explorer/Navigator} and then select \textbf{Import - General - Existing projects into Workspace}
 109 
 110 \item Select the path where you downloaded the ``umc\_dev" folder, and check all the projects: ``org.wesnoth", ``org.wesnoth.dependencies.feature", ``org.wesnoth.feature", ``org.wesnoth.ui" and ``org.wesnoth.tests". \textbf{Don't} check the "Copy projects into workspace.
 111 
 112 \item The projects will be compiled automatically. Just to be sure, as in each Java project, verify that the ``Build automatically" entry in the ``Project" menu is checked.
 113 \end{enumerate}

 126 \section{Running the tests}
 127 \subsection{Normal tests}
 128 The plugin has a Test Suite based on JUnit 4, that tests some aspects of plugin's working internals. A running instance of the plugin is not necessarily. To run these tests, follow the following steps:
 129 \begin{enumerate}
 130         \item Go to the ``Run" menu, and select ``Run configurations..."
 131         \item In the left pane's list, right click on the ``JUnit" item
 132         \item Select ``New" from the opened context menu
 133         \item Select the first radio box: ``Run a single test"
 134         \item For the \textbf{Project}, enter: ``org.wesnoth.tests" ( or browse for it )
 135         \item For the \textbf{Test Class}, enter: ``org.wesnoth.tests.WMLTestsSuite" ( or ``Search..." for it. Just enter ``WMLTestsSuite" in the newly opened window, and select the item )
 136         \item For the \textbf{Test method}, let it as it is by default: ``(all methods)"
 137         \item In the ``Test runner" Combo Box select ``Junit 4"
 138         \item Go to the ``Arguments" tab, and in the ``VM Arguments" text box, enter: $-DwesnothWorkingDir=<path>$
 139         \item Replace the $<path>$ token with the current path to the wesnoth working directory ( it contains the data directory )
 140         \item Press ``Apply" and ``Run"
 141 \end{enumerate}
 142 
 143 \subsection{PDE Tests}
 144 This tests involve starting the plugin, and launching the tests after it started. This is because some Eclipse's framework is used ( projects, etc ). To run these tests, follow the following steps:
 145 \begin{enumerate}
 146         \item Go to the ``Run" menu, and select ``Run configurations..."
 147         \item In the left pane's list, right click on the ``JUnit Plug-in Test" item
 148         \item Select ``New" from the opened context menu
 149         \item Select the first radio box: ``Run a single test"
 150         \item For the \textbf{Project}, enter: ``org.wesnoth.tests" ( or browse for it )
 151         \item For the \textbf{Test Class}, enter: ``org.wesnoth.tests.WMLPDETestsSuite" ( or ``Search..." for it. Just enter ``WMLTestsSuite" in the newly opened window, and select the item )
 152         \item For the \textbf{Test method}, let it as it is by default: ``(all methods)"
 153         \item In the ``Test runner" Combo Box select ``Junit 4"
 154         \item Go to the ``Arguments" tab, and in the ``VM Arguments" text box, add the following arguments, separated by space: 
 155         \begin{enumerate}
 156                 \item $-DisTesting=true$
 157                 \item $-DwesnothWorkingDir=<path>$ ( Replace the $<path>$ token with the current path to the wesnoth working directory ( it contains the data directory ) )
 158                 \item $-DwesnothExec=<path>$ ( Replace the $<path>$ token with the current path to the wesnoth executable )
 159                 \item $-DwesnothWMLTools=<path>$ ( Replace the $<path>$ token with the current path to the wesnoth wml tools directory )
 160                 \item $-DwesnothUserDir=<path>$ ( Replace the $<path>$ token with the current path to the user directory ( it contains the data/add-ons directory ) )
 161                 \item $-DpythonPath=<path>$ ( Replace the $<path>$ token with the current path to the python binary ( for *nix systems, 'pyhon' will suffice ) )
 162         \end{enumerate}
 163         \item Press ``Apply" and ``Run"
 164 \end{enumerate} 
 165 
 166 
 167 \textit{Note:} You can always launch last run configuration with F11 - Debug / CTRL+F11 - Run. Alternatively, you can launch them by clicking near the green bug/run icon, on the little down arrow, and selecting the run configuration.

 169 \section{Building the plugin}
 170 The plugin can be built as a zip archive (usually) for different operating systems. The built application can be launched without any Eclipse prerequisites. This plugin's version is called \textbf{Standalone}. 
 171 
 172 You now have 2 choices: Build the plugin from within Eclipse - this way you can build, say, just for your current OS/architecture - or you can use the already existing build script - that builds the plugin for all currently supported architectures.
 173 
 174 \subsection{Building via Eclipse}
 175 \begin{enumerate}
 176     \item Open Eclipse.
 177     \item Go into the ``org.wesnoth" project, and open the file ``org.wesnoth.product".
 178     \item In the bottom right part of the opened file, in section ``Exporting" there is a link: ``Eclipse Product export wizard". Click that.
 179     \item In the ``destination" group, select where you want the export/build to be done.
 180     \item Check the ``Generate metadata repository". You can also check ``Export for multiple platforms" if you want to build the plugin for other OSes/Architectures as well.
 181     \item Press Next if you have selected ``Export for multiple platforms" and select the desired OSes/Architectures.
 182     \item Finally press ``Finish" to do the building.
 183 \end{enumerate}
 184 
 185 \subsection{Building via the SCons build script}
 186 \begin{enumerate}
 187     \item Go to the ``build" directory in the repository checkout.
 188     \item The build is done via Scons.
 189     \item When launching the build first time you need to specify the ``eclipsedir" argument that specifies the path to a valid Eclipse SDK + deltapack installation. The file you have previously downloaded (``eclipse\_3.7.zip") already contains this.
 190     \item You can invoke the build like: \textit{scons  eclipsedir=/path/to/eclipse-sdk-dir}. After the first run, the path is cached and you won't need to specify it anymore.
 191     \item The directory ``wesnoth\_umc\_dev" will contain now the built plugin as zip archives, while the ``update\_site" directory will contain the update site for the plugin.
 192 \end{enumerate}

 194 \section{Generating the WML Grammar}
 195 The WML Editor uses the Xtext (\url{http://www.eclipse.org/Xtext/}) framework for getting the functionality an editor should have. The editor is based on the WML grammar defined in the ``org.wesnoth/src/org.wesnoth/WML.xtext" file.
 196 The file is written in some EBNF variant (\url{http://www.eclipse.org/Xtext/documentation.html\#\#grammarLanguage})
 197 
 198 To (re-)generate the wml grammar, right click the ``org.wesnoth/src/org.wesnoth/GenerateWML.mwe2", and select ``Run as" and after that ``MWE2 Workflow". If this is your first time running the MWE2 Workflow, you'll be prompted to download the ANTLR generator jar file. Press (y) and that will continue.

 200 \section{Frequently Asked Questions}
 201 \subsection{Where can I submit any bugs found?}
 202 Go over to Wesnoth's bug tracker: \href{https://gna.org/bugs/?func=additem&group=wesnoth&bug_group_id=116}{Add new bug}. Please be as specific as possible. Also, if you have any logs, please attach them.
 203 
 204 \subsection{Why Mac OS X 32 bit is not supported?}
 205 Apple decided to manage itself the versions that can run on each Mac version. Thus the modern Java SE6 version - which is used by the plugin - is not available on 32 bit Macs but on 64 bit only. The 32 bit Macs have just Java SE5 version available. Trying to port the plugin to the ``outdated" Java version is not likely to be done, so please consider upgrading your OS.
