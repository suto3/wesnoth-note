   5 \section{Server configuration}
   6 \label{section:communication_protocol:server}
   7 
   8 On the server there are several configuration files:
   9 \begin{itemize}
  10 \item
  11         The main configuration file, umcd.cfg, this file is stored in the
  12         data directory, which is a parameter for the programme. This file is
  13         further discussed in \cref{section:communication_protocol:server:umcd.cfg}.
  14 
  15 \item
  16         In this data directory there is a subdirectory per addon. These
  17         directories have the following files:
  18         \begin{description}
  19         \item[umc.cfg]
  20                 The config file for the umc, this file is based on the client's
  21                 pbl file. This file is further discussed in
  22                 \cref{section:communication_protocol:server:umc.cfg}.
  23 
  24         \item[umc.gz]
  25                 The gzipped umc.
  26 
  27         \item[umc-\emph{lang}.cfg]
  28                 The configuration file for the translation of the umc. The
  29                 \emph{lang} is replace with the locale of the translation.
  30                 This file is further discussed in
  31                 \cref{section:communication_protocol:server:umc-lang.cfg}.
  32 
  33         \item[umc-\emph{lang}.gz]
  34                 The gzipped mo-file for the translation. Note if the
  35                 compression-ratio for the mo-file is bad, the raw mo-file will be
  36                 stored instead. The final decision in the matter will be specified
  37                 later.
  38 
  39         \end{description}
  40 
  41 \end{itemize}
  42 
  43 
  44 \subsection{umcd.cfg}
  45 \label{section:communication_protocol:server:umcd.cfg}
  46 
  47 This file contains the settings of the umcd.
  48 
  49 \begin{lstlisting}
  50 [umcd_configuration]
  51         port = PORT
  52         threads = THREADS
  53 
  54 [/umcd_configurattion]
  55 \end{lstlisting}
  56 
  57 The keys have the following meaning:
  58 \begin{description}
  59 \item[PORT]
  60         The TCP/IP port to listen to for incoming requests.
  61 
  62 \item[THREADS]
  63         The number of listeners threads to start. A value of $0$ means start as
  64         many threads as there are detected hardware cores\footnote{Whether
  65         hyper-threading and Co. count as separate cores is unspecified.}.
  66 
  67 \end{description}
  68

  70 \subsection{umc.cfg}
  71 \label{section:communication_protocol:server:umc.cfg}
  72 
  73 This file contains the configuration of the umc on the server. The
  74 combination of this file together with the various
  75 \mbox{umc-\emph{lang}.cfg} files form the master information regarding an
  76 umc. All other files that specify the umc or its wire-protocol get their
  77 information from these files. Therefore other sections will refer to this
  78 section.
  79 
  80 The file is specified like:
  81 \begin{lstlisting}
  82 [umc]
  83         id = ID
  84         name = NAME
  85         icon = ICON
  86         type = TYPE
  87         description = DESCRIPTION
  88         version = VERSION
  89         size = SIZE
  90         authors = AUTHORS
  91         email = EMAIL
  92         uploader_ip_address = UPLOADER_IP_ADDRESS
  93         timestamp = TIMESTAMP
  94         downloads = DOWNLOADS
  95         uploads = UPLOADS
  96         password = PASSWORD
  97         language = LANGUAGE
  98         translatable = TRANSLATABLE
  99         [dependencies]?
 100                 [dependency]+
 101                         id = ID
 102                         version? = VERSION_MASK
 103                 [/dependency]
 104         [/dependencies]
 105 [/umc]
 106 \end{lstlisting}
 107 
 108 The keys have the following meaning:
 109 \begin{description}
 110 \item[ID]
 111         The unique id of the content. When uploading this value determines
 112         whether an umc is the same umc.
 113 
 114 \item[NAME]
 115         The name of the content. This string is not unique. This allows two
 116         versions of the UMC on the server. This can be used to allow a new beta
 117         next to a stable, as long as the ID differs.
 118 
 119 \item[ICON]
 120         The icon of the umc, which is shown in the addon-list on the server.
 121         TODO it is not yet specified how the icon is transferred to and from the
 122         client.
 123 
 124 \item[TYPE]
 125         The type of the addon. At the moment the following values are allowed:
 126         \begin{description}
 127         \item[CAMPAIGN\_SP] A single player campaign.
 128         \item[CAMPAIGN\_MP] A multi-player player campaign.
 129         \item[ERA] A multi-player era.
 130         \item[MAP] A multi-player map.
 131         \item[MAP\_PACK] A multi-player map pack.
 132         \item[RPG\_SP]
 133                 A RPG like single player game. (This also includes builder games in
 134                 the style of settlers.)
 135  
 136         \item[RPG\_MP]
 137                 A RPG like multi-player game. (This also includes builder games in
 138                 the style of settlers.)
 139  
 140         \item[OTHER] Everything that doesn't fit in one of the above.
 141         \end{description}
 142 
 143 \item[DESCRIPTION]
 144         The description of the umc.
 145 
 146 \item[VERSION]
 147         A string determining the version. The string should formatted like:
 148         `x.y.z' or `x.y.z-q'. When doing so the version can be parsed as version
 149         number. The `-q' part is sorted alphabetically and is an empty string if
 150         omitted, thus sorted first.
 151 
 152 \item[SIZE]
 153         The size of the umc in bytes packed on the server.
 154 
 155 \item[AUTHORS]
 156         A colon separated list of authors of the umc. When there is only one
 157         author no colon is required.
 158 
 159 \item[EMAIL]
 160         The email address of the primary author or a mailinglist. When there are
 161         problems with the umc the Wesnoth team can use this address to contact
 162         the authors, so make sure it's valid and doesn't need registration.
 163         The address is not used in other ways nor shared with third-parties.
 164 
 165 \item[UPLOADER\_IP\_ADDRESS]
 166         The ip address of the last uploader of the umc.
 167 
 168 \item[TIMESTAMP]
 169         The timestamp of the last upload. 
 170 
 171 \item[DOWNLOADS]
 172         The number of times the UMC is downloaded.
 173 
 174 \item[UPLOADS]
 175         The number of time a new version of the UMC is uploaded.
 176 
 177 \item[PASSWORD]
 178         The password required to upload a new version of an umc or remove an umc
 179         from the server.
 180 
 181 \item[LANGUAGE]
 182         The language id of the umc. If omitted American English is assumed.
 183         This allows umc authors who are not able to create content in English to
 184         still create content. However it's still recommended to try to write the
 185         content in American English. (TODO determine whether omitted on the
 186         server or only during the initial upload, if the latter the server
 187         should set it if omitted during the upload.)
 188 
 189 \item[TRANSLATABLE]
 190         This flag causes the umc to be synchronised with Wescamp. It's advisable
 191         to only set the flags when the strings are somewhat stable. This flag
 192         can only be set if the language is American English.
 193 \end{description}
 194 
 195 The [dependencies] lists the dependencies of the UMC. This allows the client
 196 to download all UMC required to use this UMC. The fields mean:
 197 \begin{description}
 198 \item[ID]
 199         The ID of the UMC to depend upon.
 200 
 201 \item[VERSION\_MASK]
 202         This mask is used to validate a version used. The format is:
 203         \begin{description}
 204         \item[$=$VERSION]
 205                 The version number of the dependency exactly match the VERSION.
 206 
 207         \item[$>=$VERSION]
 208                 The version number of the dependency must be greater or equal to
 209                 VERSION.
 210 
 211         \item[$>$VERSION]
 212                 The version number of the dependency must be greater than VERSION.
 213 
 214         \item[$<$VERSION]
 215                 The version number of the dependency must be less than VERSION
 216         \item[$<=$VERSION]
 217                 The version number of the dependency must be less than or equal to
 218                 VERSION
 219 
 220         \end{description}
 221         Several values can be used in a mask. They are separated by a space. The
 222         masks are and'ed and the user is responsible for setting a sane value;
 223         e.g. `$<$1.0.0 $>$1.0.0' can't match a version.
 224 
 225         If this field is omitted, all versions are allowed.
 226 
 227 \end{description}

 230 \subsection{umc-\emph{lang}.cfg}
 231 \label{section:communication_protocol:server:umc-lang.cfg}.
 232 
 233 The configuration file for the translation of the umc. The
 234 \emph{lang} is replace with the locale of the translation.
 235 
 236 \begin{lstlisting}
 237         [language]
 238                 id = LANGUAGE
 239                 timestamp = TIMESTAMP
 240                 translated = TRANSLATED
 241                 fuzzy = FUZZY
 242                 untranslated = UNTRANSLATED
 243                 [translated_umc_strings]
 244                         name = NAME
 245                         description = DESCRIPTION
 246                 [/translated_umc_strings]
 247         [/language]
 248 \end{lstlisting}

 252 \section{UMC configuration}
 253 \label{section:communication_protocol:umc}

 255 The pbl file contains the information regarding the addon. Its contents are
 256 stored on the system of the addon developer and on the server. The file
 257 contains the following data:
 258 
 259 \begin{lstlisting}
 260 [umc_configuraton]
 261         id = ID
 262         name = NAME
 263         icon = ICON
 264         type = TYPE
 265         description = DESCRIPTION
 266         version = VERSION
 267         authors = AUTHORS
 268         email = EMAIL
 269         password? = PASSWORD
 270         language? = LANGUAGE
 271         translatable = TRANSLATABLE
 272         [dependencies]?
 273                 [dependency]+
 274                         id = ID
 275                         version? = VERSION_MASK
 276                 [/dependency]
 277         [/dependencies]
 278 [/umc_configration]
 279 \end{lstlisting}
 280 
 281 The keys have the following meaning:
 282 \begin{description}
 283 \item[ID]
 284         See \cref{section:communication_protocol:server:umc.cfg}.
 285 
 286 \item[NAME]
 287         See \cref{section:communication_protocol:server:umc.cfg}.
 288 
 289 \item[ICON]
 290         See \cref{section:communication_protocol:server:umc.cfg}.
 291 
 292 \item[TYPE]
 293         See \cref{section:communication_protocol:server:umc.cfg}.
 294 
 295 \item[DESCRIPTION]
 296         See \cref{section:communication_protocol:server:umc.cfg}.
 297 
 298 \item[VERSION]
 299         See \cref{section:communication_protocol:server:umc.cfg}.
 300 
 301 \item[AUTHORS]
 302         See \cref{section:communication_protocol:server:umc.cfg}.
 303 
 304 \item[EMAIL]
 305         See \cref{section:communication_protocol:server:umc.cfg}.
 306 
 307 \item[PASSWORD]
 308         See \cref{section:communication_protocol:server:umc.cfg}.

 310         It may be omitted, during the initial upload. In that case the first
 311         upload results in sending back the password. It's stored in the pbl-file

 315         See \cref{section:communication_protocol:server:umc.cfg}.

 330         [request_umc_list]
 331         [/requst_umc_list]
 332 \end{lstlisting}

 335 \begin{lstlisting}
 336         [umc_list]
 337                 [umc]*
 338                         id = ID
 339                         name = NAME
 340                         type = TYPE
 341                         description = DESCRIPTION
 342                         version = VERSION
 343                         size = SIZE
 344                         authors = AUTHORS
 345                         timestamp = TIMESTAMP
 346                         downloads = DOWNLOADS
 347                         uploads = UPLOADS
 348                         language = LANGUAGE
 349                         [dependencies]?
 350                                 [dependency]+
 351                                         id = ID
 352                                         version? = VERSION_MASK
 353                                 [/dependency]
 354                         [/dependencies]
 355                         [languages]?
 356                                 [language]+
 357                                         id = LANGUAGE
 358                                         timestamp = TIMESTAMP
 359                                         translated = TRANSLATED
 360                                         fuzzy = FUZZY
 361                                         untranslated = UNTRANSLATED
 362                                         [translated_umc_strings]
 363                                                 name = NAME
 364                                                 description = DESCRIPTION
 365                                         [/translated_umc_strings]
 366                                 [/language]
 367                         [/languages]
 368                 [/umc]
 369         [/umc_list]
 370 \end{lstlisting}
 371 
 372 The fields in [umc] have the following meaning:
 373 \begin{description}
 374 \item[ID]
 375         See \cref{section:communication_protocol:server:umc.cfg}.
 376 
 377 \item[NAME]
 378         See \cref{section:communication_protocol:server:umc.cfg}.
 379 
 380         The version in [language] is the translated name of the UMC.
 381 
 382 \item[TYPE]
 383         See \cref{section:communication_protocol:server:umc.cfg}.
 384 
 385 \item[DESCRIPTION]
 386         See \cref{section:communication_protocol:server:umc.cfg}.
 387 
 388 \item[VERSION]
 389         See \cref{section:communication_protocol:server:umc.cfg}.
 390 
 391 \item[SIZE]
 392         See \cref{section:communication_protocol:server:umc.cfg}.
 393 
 394 \item[TIMESTAMP]
 395         The timestamp of the last upload. For the [umcs] it is the upload
 396         of the umc, for the [language] it is the last upload of the
 397         po-file. (NOTE MERGE WITH MASTER LIST.)
 398 
 399 \item[DOWNLOADS]
 400         See \cref{section:communication_protocol:server:umc.cfg}.
 401 
 402 \item[UPLOADS]
 403         See \cref{section:communication_protocol:server:umc.cfg}.
 404 
 405 \item[LANGUAGE]
 406         The id of a language locale. The one in [umc] is the language of
 407         the UMC. If this language is empty American English is assumed. The UMC
 408         is only translatable if this is set to American English (this should be
 409         mentioned in the PBL section).
 410 \end{description}
 411 
 412 
 413 The [dependencies] lists the dependencies of the UMC, See
 414 \cref{section:communication_protocol:server:umc.cfg}.
 415 
 416 The [languages] contains the list of languages the addon is (partly)
 417 translated to. The fields mean:
 418 \begin{description}
 419 \item[ID]
 420         The id of the language the translation for.
 421 
 422 \item[TIMESTAMP]
 423         The timestamp the po-file was last updated. NOTE UNIX stamp, should be
 424         at other stamp as well.
 425 
 426 \item[TRANSLATED]
 427         The number of translated strings, if this entry is $0$, then the entire
 428         section is not printed.
 429 
 430 \item[FUZZY]
 431         The number of strings that are fuzzy.
 432         
 433 \item[UNTRANSLATED]
 434         The number of strings that are not translated.
 435 
 436 NOTE MAYBE ALSO ADD UP AND DOWNLOAD COUNTER and obviously the SIZE
 437 \end{description}
 438 
 439 The section [translated\_umc\_strings] duplicates some strings from the
 440 general [umc] section. This version contains the translated string from
 441 the po file. If not translated or fuzzy it contains the original, just like
 442 normal with gettext. The keys in this section corrospondent with the ones in
 443 [campaign].

 445 \section{Request umc download}
 446 \label{wire:request_umc_download}
 447 
 448 This requests to download an umc.
 449 
 450 Request:
 451 \begin{lstlisting}
 452         [request_umc_download]
 453                 id = ID
 454                 [languages]?
 455                         [language]*
 456                                 ID = LANGUAGE
 457                         [/language]
 458                 [/languages]
 459         [/requst_umc_download]
 460 \end{lstlisting}
 461 
 462 The fields are:
 463 \begin{description}
 464 \item[ID]
 465         The id of the umc to download.
 466 
 467 \item[LANGUAGE]
 468         The id of the requested language to download.
 469 
 470 \end{description}
 471 
 472 Replay:
 473 \begin{lstlisting}
 474         [umc_download]
 475                 id = ID
 476                 status = STATUS
 477                 [languages]?
 478                         [language]+
 479                                 id = LANGUAGE
 480                                 status = STATUS
 481                         [/language]
 482                 [/languages]
 483         [/umc_download]
 484 \end{lstlisting}
 485 
 486 This is a mirror of the request, only all items have a status field. The
 487 status can be OK it the file exists and can be downloaded. Or ERROR if not
 488 possible.
 489 
 490 The files send are in the order of appearance in the request and reply. The
 491 files are named:
 492 \begin{description}
 493 \item[ID.gz]
 494         The gzipped campaign files. The file is packed in a single config file
 495         to be extracted to the local file-system.
 496 
 497 \item[ID-LANGUAGE.gz]
 498         A gzipped mo-file for a language. If gzip's compression ratio for mo
 499         files is too small the mo-file will be send directly.
 500 
 501 \end{description}
